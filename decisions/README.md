### 2026-03-27: Set up 4 GitHub repos

**Decision:** Create 4 separate repos (`second-brain`, `ai-blueprints`, `daily-commits`, `knowledge-vault`) instead of one mono-repo.

**Alternatives considered:**
1. One big mono-repo with subdirectories
2. Two repos (personal notes + technical content)
3. Four separate repos (chosen)

**Reasoning:**
- Each repo has a distinct purpose and audience
- Separate repos = separate commit histories = each repo's graph tells its own story
- Easier to share individual repos without sharing everything
- GitHub topics/descriptions work better per-repo

**Potential downsides:**
- More repos to maintain
- Cross-referencing between repos is less clean
